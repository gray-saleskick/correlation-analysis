SECTION 3: BACKEND SECURITY FIXES
===============================================================================

3.1 PATH TRAVERSAL PROTECTION (lib/store.ts) [CRITICAL]
  - Added isValidId() function: validates IDs match /^[a-z0-9][a-z0-9_-]{0,60}$/
  - clientDir() now validates clientId before constructing path
  - Added resolved path check: ensures path stays within CLIENTS_DIR
  - listClients() now skips hidden directories and validates entry names

3.2 INPUT VALIDATION (All API Routes) [CRITICAL]
  - Client name: max 100 characters, trimmed
  - Application title: max 200 characters, trimmed
  - Application count: max 50 per client
  - PUT body size: max 10MB via content-length check

3.3 ERROR HANDLING (All API Routes) [HIGH]
  - Wrapped all route handlers in try/catch blocks
  - Generic "Internal server error" messages returned to client
  - Detailed errors logged server-side via console.error
  - No stack traces or internal paths leaked to client

3.4 CONTENT-TYPE VALIDATION (POST /api/clients) [HIGH]
  - POST requests now validate Content-Type: application/json
  - Returns 415 Unsupported Media Type for non-JSON requests

3.5 ATOMIC FILE WRITES (lib/store.ts) [MEDIUM]
  - writeProfile() now writes to .tmp file first, then renames
  - Prevents data corruption from interrupted writes
  - Profile file is never in a partially-written state

3.6 SECURITY HEADERS (next.config.mjs) [MEDIUM]
  - X-Content-Type-Options: nosniff
  - X-Frame-Options: DENY (clickjacking protection)
  - X-XSS-Protection: 1; mode=block
  - Referrer-Policy: strict-origin-when-cross-origin
  - Permissions-Policy: camera=(), microphone=(), geolocation=()
  - API routes: Cache-Control: no-store, no-cache, must-revalidate

3.7 FILE SIZE LIMIT (lib/store.ts) [MEDIUM]
  - readProfile() now checks file size (max 50MB)
  - Prevents memory exhaustion from corrupted/malicious profile files

3.8 COLLISION LIMIT (lib/store.ts) [LOW]
  - createClient() now limits ID collision retries to 1000
  - Prevents infinite loop from adversarial input

SECTION 4: KNOWN ISSUES & RECOMMENDATIONS FOR FUTURE
===============================================================================

4.1 AUTHENTICATION (Not Implemented — Needed Before Production)
  - Currently NO authentication on any API route
  - Recommendation: Add JWT or session-based auth middleware
  - All routes are publicly accessible

4.2 CSRF PROTECTION (Not Implemented)
  - State-changing operations (POST/PUT/DELETE) lack CSRF tokens
  - Recommendation: Add origin/referer validation middleware

4.3 RATE LIMITING (Not Implemented)
  - No rate limiting on any endpoint
  - Recommendation: Use upstash/ratelimit or similar

4.4 TYPEFORM PAT STORAGE
  - Still stored in plain text in profile.json
  - Recommendation: Use environment variables or encrypted secrets vault

4.5 NATIVE confirm() DIALOGS
  - Several confirm() calls remain in ApplicationDetail.tsx sub-components
  - showConfirm() function is available but not yet wired to all locations
  - Recommendation: Migrate remaining confirm() calls to custom modal

4.6 NEXT.JS VERSION
  - Currently on 14.2.35 (has known vulnerabilities)
  - Recommendation: Upgrade to latest stable when ready
